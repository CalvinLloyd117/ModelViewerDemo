Annotations and Hotspots

Model-Viewer allows Hotspots to be (relatively) easily inserted into the 3D space.

To define a hotspot, buttons must be defined between the <model-viewer> tags. These buttons must follow a particular naming convention (ex. slot="hotspot-etching"). Additionally, the positions of the buttons must be defined using a 3D coordinate system. Finally, the data-normal property of the buttons must also be defined. This is how button visibility is managed.

These hotspots are for all intents and purposes, css buttons.

Therefore, all css tricks such as hover, visibility, on-click, etc. are all applicable to them.

Furthermore, all JavaScript and jQuery methods can also be used on them, such as checking for attributes and states (like visibility).

NOTE: This feature is only present in an unreleased development version of Model-Viewer.
